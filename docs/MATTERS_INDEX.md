# Matters index: storage and rebuild

## Stored preparation (no per-query work)

All preparation for search is **stored in the `matters` table** and reused:

- **`embedding`** — Vector (pgvector) computed once when the row is inserted or updated, used for semantic search. Queries only read it.
- **`search_vector`** — `tsvector` generated by Postgres from `client_name`, `matter_name`, `matter_description`, `practice_area`, `matter_type`, and `related_keywords`. Stored and indexed; queries only read it.

Nothing is recomputed on each suggestion request; the API just queries the table.

## When data changes: rebuild

When the source dataset changes (e.g. you edit `data/matters.json` or load from another source), rebuild the index so stored embeddings and search vectors match the new data.

### Option 1: Rebuild from JSON (recommended)

From the project root, with `DATABASE_URL` set (e.g. in `.env`):

```bash
# Full rebuild: drop table, recreate with current schema, load from data/matters.json
python -m app.seed_matters --rebuild

# Rebuild from a specific file
python -m app.seed_matters --rebuild path/to/matters.json
```

Use this when the JSON file is the source of truth and you want the table to reflect it exactly. Also run `--rebuild` after the Matters table schema changes (e.g. new columns) so the table is recreated with the new structure.

### Option 2: Upsert only (no truncate)

```bash
# Insert or update rows from data/matters.json (no truncate)
python -m app.seed_matters
```

Existing rows with the same `(client_id, matter_id)` are updated (including embeddings); new rows are inserted. Use when you’ve only added or changed a subset of records.

### Option 3: App startup

On startup, the app runs a background task that seeds from `data/matters.json` using the same upsert logic as Option 2. Restarting the app after changing the JSON will refresh those rows on the next seed run.

## Summary

| Goal                         | Command / action                          |
|-----------------------------|-------------------------------------------|
| Stored, reused at query time | Automatic (table columns + indexes)      |
| Full rebuild when data changes | `python -m app.seed_matters --rebuild`  |
| Incremental update          | `python -m app.seed_matters` or restart app |
